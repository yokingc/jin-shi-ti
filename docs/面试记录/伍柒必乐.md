面前准备：
React Router 主要用于 SPA 的页面切换和路由管理。实际项目里我一般会用它做页面路径映射、动态路由参数获取、嵌套路由拆分，以及配合懒加载做按路由维度的资源加载。像列表页跳详情页，我会通过 useNavigate 跳转，通过 useParams 或 search 参数拿业务 ID。

redux：
单向数据流：组件 dispatch action → reducer 处理 → store 状态更新 → 组件重新渲染
核心思想是：
全局状态放在一个 store 里
不能直接改状态
必须通过 action 描述“要做什么”
reducer 根据 action 返回新状态

react-reduex：
1）useSelector从 store 里取数据
2）useDispatch派发 action
3）Provider把 store 注入应用

redux / React Redux / Redux Toolkit的对比

less 和 scss对比：
它们都是 CSS 预处理器，本质上都是为了让 CSS 更好维护，支持变量、嵌套、混入、函数等能力。
Less 用 @ 定义变量，SCSS 用 $
SCSS 在语法和生态上通常更接近现代前端项目
Less 在一些 UI 库生态里更常见，比如以前的 Ant Design Less 体系
两者本质都是为了增强 CSS 可维护性

高频追问：
1.React Router 底层原理你知道多少：1.监听路由变化。2.维护当前 location 状态。3.做路由匹配并渲染对应组件。
BrowserRouter 依赖的是 H5 History API，HashRouter 依赖的是 URL hash 变化，也就是 # 后面的内容变化不会真正请求服务端。
2.Redux 单向数据流指的是：view 触发 action → dispatch 到 store → reducer 计算新 state → view 基于新 state 重新渲染。让状态变化可预测、可追踪、可调试。
3.Redux 和 Context 区别：都能做跨层级数据传输，但定位不同。conText适合比较稳定、层级较深、逻辑不复杂的问题。redux是更完整的状态管理方案，除了共享数据还提供统一store，更新粒度和规范，中间件，devtools处理。
4.useMemo和useCallback区别：useMemo缓存函数结果，useCallback缓存函数本身，实践中经常和react.memo使用。不能滥用。
5.Webpack loader 和 plugin 区别：loader 本质上是 文件转换器，webpack本身只认js和json。plugin 是 插件机制，介入webpack的整个打包生命周期，提供功能扩展。
6.Babel 做了什么转换：Babel 本质上是 JavaScript 编译器。Babel 负责语法降级如ES6、jsx、TS等，polyfill 负责补运行时缺失能力，比如一些数组能力。
